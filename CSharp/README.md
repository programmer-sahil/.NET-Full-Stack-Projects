
## C# Cheat Sheet Beyond Basics

This section skips variables, operators, data types, and basic if-else.

### Program Structure

```csharp
using System;

class Program
{
    static void Main()
    {
        Console.WriteLine("Hello");
    }
}
```

Important idea:

- `Main()` is the entry point for a traditional console app.
- Methods marked `static` can be called without creating an object.
- Keep related logic in separate methods to make code readable.

### Method Patterns

```csharp
static int Add(int a, int b)
{
    return a + b;
}

static void PrintMessage(string message)
{
    Console.WriteLine(message);
}

static bool IsEven(int number)
{
    return number % 2 == 0;
}
```

Remember:

- Use `void` when the method does not return anything.
- Use return types like `int`, `string`, `bool`, `List<int>` when the method returns data.
- Small methods are easier to test and debug.

### Arrays

```csharp
int[] numbers = { 10, 20, 30 };

for (int i = 0; i < numbers.Length; i++)
{
    Console.WriteLine(numbers[i]);
}
```

Common interview patterns:

- Traverse from left to right.
- Traverse from right to left.
- Compare every pair with nested loops.
- Track `largest`, `smallest`, `count`, `sum`, or `index`.
- Use a separate result array when you want brute-force easy logic.

### Strings

```csharp
string text = "hello";

for (int i = 0; i < text.Length; i++)
{
    Console.WriteLine(text[i]);
}
```

Useful members:

```csharp
text.Length
text.ToUpper()
text.ToLower()
text.Contains("he")
text.StartsWith("h")
text.EndsWith("o")
text.Replace("h", "H")
text.Trim()
text.Split(' ')
```

Interview patterns:

- Reverse by reading characters from the end.
- Palindrome by comparing left and right indexes.
- Frequency count by nested loops or dictionary.
- Anagram by sorting, counting, or brute-force matching.
- Remove spaces by building a new string.

### Dictionary

Use `Dictionary<TKey, TValue>` for fast lookup by key.

```csharp
Dictionary<char, int> frequency = new Dictionary<char, int>();

foreach (char ch in "hello")
{
    if (frequency.ContainsKey(ch))
    {
        frequency[ch]++;
    }
    else
    {
        frequency[ch] = 1;
    }
}
```

Useful for:

- Character frequency
- Word count
- Duplicate detection
- Fast lookup
- Mapping IDs to objects

Common LINQ methods:

### Classes And Objects

```csharp
class Student
{
    public int Id { get; set; }
    public string Name { get; set; } = "";

    public void Print()
    {
        Console.WriteLine(Id + " " + Name);
    }

student.Id = 1;
```

Remember:

- Class is a blueprint.
- Object is an instance of a class.
- Properties store object data.
- Methods define object behavior.

### Exception Handling

```csharp

{
    int number = int.Parse("123");

{

finally
{
```

Best practice:

- Catch specific exceptions.
- Do not hide errors silently.
- Log exceptions in real applications.


Lifetimes:

- `AddTransient`: new object every time.

- `AddSingleton`: one object for the whole application.

## Interview Practice Checklist

Array problems:

- Find largest element
- Find smallest element
- Find second largest element
- Reverse an array
- Linear search
- Count even and odd numbers
- Find duplicates
- Remove duplicates
- Move zeroes to end
- Check if two arrays are equal
- Merge two arrays

String problems:

- Reverse string
- Check palindrome
- Count vowels
- Count words
- Remove spaces
- Find duplicate characters
- Character frequency
- Check anagram
- Convert lowercase to uppercase manually
- Find first non-repeating character

Brute-force approach:

- Read the problem carefully.
- Write input and expected output.
- Use loops first.
- Use nested loops when comparing every element with every other element.
- Use extra array or string when that makes the logic easier.
- After brute force works, then optimize.
